Error Log Screen
Screen Heading: Logs > Error
Use the Error Log screen to view entries in the Error log (WMERROR_yyyymmdd_hhmmss.log). The Integration Server error log maintains information about service exceptions. When it records an exception in the log, the server also includes a stack trace showing the sequence of calls leading to the exception. For information about changing the display of this log, see webMethods Audit Logging Guide.
To open the Error Log screen, click Error under the Logs menu on the Navigation panel.
Log display controls
Display Log Entries oldest to the newest starting from the beginning
View entries in the log from the oldest to the most current.
Display Log Entries newest to the oldest starting from the end
View entries in the log from the most current to the oldest.
Number of entries to display
By default, the Integration Server Administrator displays the 35 most recent entries in the log. You can change the number of entries displayed.
Tip:
If you have Administrator privileges, you can permanently change the default value of this field. Go to the Settings > Extended page and modify the watt.server.log.maxEntries parameter.
From/To
Specify, in YYYY-MM-DD HH:MM:SS date format, the range of dates and times for which you want log entries to be displayed.
Note:
The From/To fields are displayed only if the logger associated with this log is configured to write to a database. For more information, see the webMethods Audit Logging Guide.
Expand Stack Trace Data
Display the expanded form of the stack trace data.
Refresh
Click this button to update the display after changing the log display controls.
Tip:
If you have Administrator privileges, you can change the default refresh interval for log entries (the default is 90 seconds). Go to the Settings > Extended page and modify the watt.server.log.refreshInterval.
Error Log Entries as of mm-dd hh:mm:ss
Time Stamp
Date and time the entry was written to the log.
Service Name
Name of the service in which the error occurred.
Service Stack
Parent services for the service in which the error occurred.
Tip:
If you have Administrator privileges, you can change the level of detail that the Integration Server Administrator displays for logged exceptions. Go to the Settings > Extended page and modify the watt.server.deprecatedExceptionLogging parameter. For more information about this parameter and an explanation of the format of the Service Stack, see webMethods Integration Server Administratorâ€™s Guide.
Error Message
Message describing the error that occurred.
Stack Trace
Trace that shows the call sequence leading to the exception. To expand this display, check Expand Stack Trace Data and click Refresh.
Root Context, Parent Context, Current Context
Context information that provides a consistent, interpretable audit trail of activity that the webMethods Monitor can use to relate entries within the log.

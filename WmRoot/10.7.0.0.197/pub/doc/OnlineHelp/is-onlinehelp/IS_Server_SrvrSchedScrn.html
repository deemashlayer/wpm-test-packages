Scheduler Screen
 
View System Tasks Screen
Create a Scheduled Task Screen
Screen Heading: Server > Scheduler
Use the Scheduler screen to view the list of scheduled services and remove services from the schedule.
Integration Server automatically deletes the expired scheduled services from the Scheduler screen. If you do not want Integration Server to delete the expired tasks until the next server restart, set the watt.server.scheduler.deleteCompletedTasks parameter to false.
To open the Scheduler screen, click Scheduler under the Server menu on the Navigation panel.
Click Pause Scheduler (currently Running) to pause the execution of scheduled tasks.
Click Resume Scheduler (currently Paused) to resume the paused execution of scheduled tasks.
Click View system tasks to display the list of scheduled system tasks.
Click Create a Scheduled Task to schedule a task.
Click Filter Services to limit the display of services that are scheduled to run.
Click Show All Services to disable filtering and restore the default list of all scheduled services. This also updates the lists of scheduled tasks to reflect any changes that occurred to the Integration Server (such as new tasks) while Filter Services was enabled.
Filtering
Enter Complete/Part of Service Name
The string you want to submit to the filter. Filter criteria can be literals or a combination of literal and wild-card characters. The "*" (asterisk) and "?" (question mark) are the only supported wild-card characters. Leaving the filter criteria blank includes all services.
Important:
The service names in the Service Name field are case-sensitive. For example, if you enter "Wm*", the filter will ignore any service that begins with "wm".
Status Is Active
Controls whether only active services are displayed. If not checked, services with all statuses are displayed.
Hide Remote Tasks
Controls whether only tasks that run on the current server are displayed. This also includes tasks scheduled to run on the server when the Target Node is set to run on All servers in the cluster or Any server connected to the database. Tasks scheduled to run only on other servers are not displayed.
Submit
Processes the filter criteria and updates the page.
One-Time and Simple Interval Tasks
This section displays tasks that are scheduled to execute a single time or at a simple repeating interval. For simple interval tasks the server continues to execute the service until you cancel the scheduled user task, or if the user task does not persist after restart, until the server is restarted.
ID
Displays a task ID that uniquely identifies this task.
Service
Displays the name of the service that has been scheduled to execute. To edit the schedule for this service, click the name of the service.
Description
A description of the task.
Queue Name
For Trading Networks, the name of the queue associated with a scheduled delivery task.
Last Error
The last error that the task encountered.
Run As User
Displays the user name under which the scheduled service runs (e.g., Administrator).
Target
Server or servers on which the task is to run. This field can contain the host name of a particular Integration Server, Any Server (the task will run on any, but only one, server connected to the same), or All Servers (the task will run on all servers in a cluster). If the field displays a particular server, and a logical server name has been defined for that server on the watt.server.scheduler.logical.hostname parameter, this field displays the logical server name.
Interval
Displays the frequency at which the service runs. Displays "once" if the service is scheduled to execute a single time. For a repeating task, this field shows the number of seconds that the server waits between executions of the service. For example, if you want the server to execute the service every 24 hours, it will show 86400 seconds.
Next Run
Specifies the number of seconds until the next execution of the service. Next Run value will be "N/A" for the parent node when a task is scheduled to run on all cluster nodes.
Status
Indicates whether this scheduled task is active or suspended. To temporarily suspend a scheduled task, click the Active link. If a task is suspended, it will not execute at its scheduled times. To reactivate a suspended task, click the Suspended link.
Remove
Deletes the task from the schedule. To delete a task, click the icon in the Remove column.

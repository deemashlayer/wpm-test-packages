Edit Password Restrictions Screen
Screen Heading: Security > User Management > Password Restrictions > Edit
Use the Edit Password Restrictions screen to edit the criteria for passwords. You can also use this screen to specify whether users are allowed to change their own passwords.
To open the Edit Password Restrictions screen, go to the User Management Screen, click Password Restrictions, and then click Edit Password Restrictions.
Click Return to User Management to return to the list of groups and users that are currently defined on your server.
Click Return to Password Restrictions to view the password settings.
Password Restrictions
Use this section to specify whether users are permitted to change their passwords and to specify the criteria that the passwords they specify must satisfy.
Enable Password Change
Specifies whether non-Administrator users are allowed to change their passwords. If you select No, users who are not members of the Administrators ACL must have a system administrator change the passwords for them. The default is Yes.
Password Enforcement Mode
Specifies whether Administrator users are allowed to choose passwords that are not impacted by the password restrictions setting. If the value is set to Lax, Administrator users can choose passwords that do not conform to the password restrictions setting. If the value is set to Strict, Administrator users must choose passwords that follow the password restriction settings. For non-administrators the restrictions are always enforced. The default is Lax.
Minimum Password Length
Specifies the minimum number of total characters a password must contain. The default is 8.
Maximum Password Length
Specifies the maximum number of total characters a password can contain. Maximum number of total characters a password can contain is 128. The default is 64 characters.
Minimum Number of Uppercase Characters
Specifies the minimum number of uppercase alphabetic characters the password must contain. The default is 0.
Minimum Number of Lowercase Characters
Specifies the minimum number of lowercase alphabetic characters the password must contain. The default is 0.
Minimum Number of Digits
Specifies the minimum number of numeric digits the password must contain. The default is 0.
Minimum Number of Special Characters (neither alphabetic nor digits)
Specifies the minimum number of special characters, such as asterisk (*), period (.), question mark (?), and ampersand (&), the password must contain. The default is 0.
Maximum Number of Identical Characters in a Row
Specifies the maximum number of continuous identical characters a password can contain. The default is 3.
Number of Old Passwords to Remember (per user)
Specifies the maximum number of previously set passwords that Integration Server remembers for a user. You cannot choose a password that matches with any of the passwords stored in memory. Maximum number of passwords saved is 12. The default is 0.
Note:
Setting a new value to this field takes effect immediately upon saving. All the previously saved passwords, corresponding to the old value set for this field, are deleted and cannot be retrieved.
For example, If initially administrator sets the value of this field to five, then Integration Server saves the last five passwords. If the administrator changes the value of this field to three, then Integration Server maintains only the latest three password entries and removes the oldest two entries. Similarly, if administrator changes it to six, then Integration Server maintains six latest entries.
Save Password Settings
Saves the password settings that you have specified. The changes take effect immediately.
Note:
These changes do not affect passwords that already exist on the server. The new requirements will be imposed on those passwords that users define from this point forward only.

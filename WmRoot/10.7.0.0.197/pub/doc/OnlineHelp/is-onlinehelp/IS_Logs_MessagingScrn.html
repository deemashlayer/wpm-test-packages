Messaging Log Screen
Screen Heading: Logs > Messaging
Use the Messaging Log screen to view the contents of the most recent messaging audit log (WMMESSAGING_yyyymmdd_hhmmss.log). The messaging audit logger writes to the messaging log when enhanced logging is configured for a Universal Messaging connection alias or a JMS connection alias and the destination for the log entries is the messaging log.
To open the Messaging Log screen, click Messaging under the Logs menu on the Navigation panel.
Log Display Controls
Display Log Entries oldest to newest starting from the beginning
View entries in the log from oldest to the most current.
Display Log Entries newest to the oldest starting from the end
View entries in the log from the most current to the oldest.
Number of entries to display
By default, the Integration Server Administrator displays the 35 most recent entries in the log. You can change the number of entries displayed.
Tip:
If you have Administrator privileges, you can permanently change the default value of this field. Go to the Settings > Extended page and modify the watt.server.log.maxEntries parameter.
UUID
The UUID (Universally Unique Identifier) for which you want to view log entries.
Refresh
Click this button to update the display after changing the log display controls.
Tip:
If you have Administrator privileges, you can change the default refresh interval for log entries (the default is 90 seconds). Go to the Settings > Extended page and modify the watt.server.log.refreshInterval.
Messaging Log Entries as of yyyy-mm-dd hh:mm:ss
Time Stamp
Date and time the entry was written to the log.
Mode
Indicates whether Integration Server wrote the log message on behalf of a message producer or consumer for webMethods messaging or JMS messaging.
Message
Text of the logged message.
UUID
Universally unique identifier for the message being sent or received. In webMethods messaging, you can set a value for the _env/uuid field when publishing the document. If you do not set a UUID value, Integration Server assigns one during publication. For JMS messaging, you can set a UUID in the JMSMessage/properties/uuid field when sending the JMS message. For JMS messaging, the uuid field is optional. Integration Server does not generate or assign a UUID if one is not specified.
Alias
Name of the messaging connection alias associated with the sent or received message. The messaging connection alias can be a Universal Messaging connection alias or a JMS connection alias.
Destination
Destination to which the message was sent. For webMethods messaging, the Destination column contains the name of the channel to which the message was sent. For JMS messaging, the Destination column contains the name of the queue or topic to which the JMS message was sent.
Message ID
The ID assigned to the message by the messaging provider. For webMethods messaging, the Message ID is the Universal Messaging event ID which Universal Messaging generates when it first receives the message. For JMS messaging, the Message ID is the JMSMessageID which is generated by the JMS provider.
Trigger
Name of the trigger that received the message.

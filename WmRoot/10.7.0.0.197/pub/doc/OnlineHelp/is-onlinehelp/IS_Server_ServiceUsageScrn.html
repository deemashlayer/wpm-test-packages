Service Usage Screen
Screen Heading: Server > Service Usage
Use the Service Usage screen to view statistics for services that the webMethods Integration Server has executed since it was last started. This screen also includes information about cache usage, so you can monitor the effectiveness of your cache settings.
To open the Service Usage screen, click Service Usage under the Server menu on the Navigation panel.
Click Reset Server Cache to clear the cache for all services.
Select Show running services on top to move all currently running services to the top of the service list. (Deselect it to restore the list to alphabetical order.) Running services are identified by a number in brackets, at the right of the service name.
Service
The fields in this section display the list of services executed since the Integration Server was started.
Name
Displays the name of the service. A bracketed number to the right of the service name identifies how many instances of the service, if any, are currently running.
Count
Displays the number of times that the service has executed since the server was last started.
Last Ran
Displays the date and time the service was last executed.
Cached
Indicates whether the service's caching option is enabled. To enable/disable the cache setting for a service, you use Software AG Designer.
Prefetched
Indicates whether the service's prefetch option (for caching) is enabled. To enable/disable the prefetch setting for a service, you use Software AG Designer.
Cache and Prefetch Information
The fields in this section display information about cache utilization for those services that are currently using cache.
Name
Displays the name of the service.
Last Cache Hit
Displays the date and time that a cached result for this service was last accessed (hit).
Cache Hits
Displays the total number of times that a cached result for this service has been accessed (hit).
Cache Hits/Total Count
Displays the percentage of service requests that have been fulfilled by retrieving the results from cache.
Cache Entries
Displays the number of active cache entries for the service.
Cache Expires
Displays the number of expired cache entries for the service.
Last Prefetch
Displays the date and time that the server last prefetched and cached the results for the service.
Note:
The cache may not be refreshed at the exact time specified in Cache Expires. It may vary from 0 to 15 seconds, according to the cache sweeper thread. Select watt.server.cache.flushMins setting on the Show and Hide Keys Screen to enable the display of the actual cache refresh time on the Extended Settings Screen.
Total Prefetches
Displays the total number of times the server has prefetched and cached the results for the service.
Recent Prefetch
Displays the total number of times the server has prefetched and cached the results for the service in the last poll interval. (The poll interval is set to 10 minutes on most servers.)
Circuit Breaker Information
Integration Server gathers circuit breaker statistics for each service with a configured circuit breaker.
Note:
The circuit breaker feature is available by default for a service that resides on a Microservices Runtime. To use the circuit breaker feature with Integration Server, your Integration Server must have additional licensing.
Note:
If a circuit breaker is not configured for any service, the Circuit Breaker Information table displays the message “No Services with a Circuit Breaker Enabled”.
Name
Name of the service for which a circuit breaker is configured.
State
The state of the circuit. The circuit state can be one of the following:
Closed. The service executes.
Open. The service does not execute. Instead, depending on the circuit breaker configuration, the service returns an exception or executes an alternative service.
Half-Open. The first request for this service since the circuit state changes to half-open results in service execution. All other requests wait. If the service executes successfully, the circuit is closed and waiting requests execute. If the service ends with a failure exception, the circuit is re-opened.
Open Time
Time at which circuit breaker last set the circuit state to open.
Half-Open Time
Time at which circuit breaker last set the circuit state to half-open.
Closed Time
Time at which the circuit breaker last set the circuit state to closed.
Open Count
Number of times that circuit breaker set the circuit state to open since Integration Server started
Request Count
Number of incoming requests for the service since the circuit breaker changed the circuit state to open.

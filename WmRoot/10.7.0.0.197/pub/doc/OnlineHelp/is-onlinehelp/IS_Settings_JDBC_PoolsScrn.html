Screen Heading: Settings > JDBC Pools
You use the JDBC Pools screen to connect Integration Server and products that run on Integration Server to database components. A database component is a grouping of database objects.
To open the JDBC Pools screen, click JDBC Pools under the Settings menu on the Navigation panel.
Click Create a Pool Alias Definition to define connections to a database.
Click Copy a Pool Alias Definition to copy the settings of an existing pool alias to create a pool alias.
Click Create a Driver Alias Definition to identify a database driver.
Click Edit to change the definition of a specific functional, pool, or driver alias.
Click the icon in the Test column to validate the connection to the database, using the connection information defined in the pool alias.
Click Restart to refresh the connection information for the database.
Functional Alias Definitions
A functional alias tells Integration Server or products that run on Integration Server to write data to a particular database component. Integration Server provides predefined functional aliases that correspond to most database components (for example, IS Core Audit Log function that corresponds to the ISCoreAudit database component, a Cross Reference function that corresponds to the CrossReference database component, and so on). You direct each functional alias to write to its database component by pointing the functional alias at a JDBC database connection pool.
Pool Alias Definitions
Integration Server uses JDBC database connection pools to connect to database components. Each pool defines the connection to a database server that hosts database components.
Driver Alias Definitions
JDBC database connection pools use JDBC database drivers to connect to database components.

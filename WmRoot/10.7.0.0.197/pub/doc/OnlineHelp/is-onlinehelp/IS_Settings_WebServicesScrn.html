Screen Heading: Settings > Web Services
Use the Web Services Settings screen to view all web services endpoints on Integration Server.
To open the Web Services Settings screen, click Web Services under the Settings menu on the Navigation panel.
Click Reliable Messaging to view the reliable messaging sequence reports.
Click Create Web Service Endpoint Alias to create a new alias.
Click Set Default Provider Endpoint Alias to set the default provider web service endpoint alias for HTTP and/or HTTPS.
Web Service Endpoints List
Column Name
Description
Default
Indicates which provider web service endpoint alias is the default for the Integration Server. You can have a default alias for the HTTP and HTTPS protocols for provider web service endpoint aliases. You cannot set a default alias for JMS or for consumer web service endpoint aliases.
Alias
The name assigned to the endpoint alias. Click on the name to view the values of the alias.
Description
Description for the endpoint alias.
Delete
Deletes the web services endpoint from the Integration Server. If the port is in use when it is deleted, the following behavior occurs:
For a provider service, the port reverts to the default localhost value.
For a consumer service, the port reverts to the value defined in the WSDL.
Note:
You cannot delete the default provider endpoint alias.
Note:
You cannot delete a message addressing endpoint alias if a web service endpoint alias for provider web service descriptor is using the message addressing endpoint alias as a part of its response map. For more information about response map, see Create Web Service Endpoints Alias Screen.

Edit Subscription Screen
Screen Heading: Packages > Subscribing > Edit Subscription
Use the Edit Subscription screen to change information for a subscription on a remote server. You can edit only those subscriptions that your server initiated. To update a subscription that was initiated by the publisher, contact the administrator of that server.
To open the Edit Subscription screen, go to the Subscribing Screen and click Update and Unsubscribe From Remote Package. The Update and Unsubscribe from Remote Package Screen will be displayed. In the row with the subscriber name, click the Edit link in the Update column.
Note:
The publishing server must be running when you edit the subscription. Your server will connect to it during this process.
Click Return to Subscribing to return to the list of package subscriptions.
Package
Specifies the name of the package to update. Be sure to type the name exactly as it is specified on the publishing server, using the same combination of uppercase and lowercase characters.
Package
Specifies the name of the package to update. Be sure to type the name exactly as it is specified on the publishing server, using the same combination of uppercase and lowercase characters.
Publisher Alias
Specifies the publisher's alias. An alias is a named set of parameters that tells your server how to connect to the publisher when you want to retrieve (pull) this package. It specifies connection information such as the host name and port number. If you have not already defined an alias for this publisher, click the Publisher Alias link to access the Subscribe to Remote Package Screen.
Local Port
Specifies the port number on which you want your server to receive this package when the remote server sends (pushes) this package to your server.
Local User Name
Specifies the user that the remote server will use when it sends (pushes) this package to your server. This user must belong to a user group that is assigned to the Replicators ACL on your server.
Local Password
Specifies the password for the user name in Local User Name.
Notification Email
Optional. Specifies the e-mail address of the user who will be notified each time the publisher creates a release of this package. (To use this option, you must also set the SMTP Server option on the Resources Screen to specify the address of the SMTP server that your server will use to receive e-mail messages.)
Automatic Pull
Optional. Specifies whether your server will retrieve this package automatically when the publisher issues a new release. When you enable Automatic Pull, the publisher sends an e-mail message to your server when it releases a new version of this package. When your server receives this message, it logs on to the publisher's server and retrieves the package automatically. If you prefer to retrieve this package manually, disable the Automatic Pull option.
Important:
If you want to use the Automatic Pull feature, you must configure an SMTP port on your server. For information about configuring ports, see webMethods Integration Server Administrator’s Guide.
Automatic Pull Email
Required if Automatic Pull is enabled. Specifies the e-mail address to which the publisher is to send the message that triggers the automatic pull process. You must specify an email address on your webMethods Integration Server that is reserved exclusively for this purpose. Do not use the same address as Notification Email.
Submit Changes
Submits the requested changes to the publisher.

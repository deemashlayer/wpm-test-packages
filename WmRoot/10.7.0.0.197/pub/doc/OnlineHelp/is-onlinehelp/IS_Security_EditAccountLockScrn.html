Edit Account Locking Settings Screen
Screen Heading: Security > User Management > Account Locking Settings > Edit
Use the Edit Account Locking Settings to specify the account locking criteria for users. Integration Server locks a user account when a user fails to provide the correct password after a specified number of attempts. You also use this screen to specify whether to enable the user account locking settings.
To open the Account Locking Settings screen, go to the User Management Screen, click Account Locking Settings, and then click Edit Account Locking Settings.
Click Return to User Management to return to the list of groups and users that are currently defined on your server.
Click Return to Account Locking Settings to view the account locking settings of users.
Click Locked Users to view the list of users whose Integration Server accounts are locked.
Edit Account Locking Settings
Use this section to specify the account locking settings for all users. You also use this screen to specify whether to enable the account locking settings.
Enabled
Specifies whether to enable the account locking settings. Select Enabled to enable the password expiry settings.
Maximum Login Attempts
Specifies the number of attempts in the specified time interval (minutes, hours, or days) to provide the correct password before Integration Server locks the account.
Lockout Duration
Specifies the duration for which the account remains locked. Integration Server unlocks the user account after the block duration elapses.
Note:
If the value is set to 0, then the account does not unlock automatically. Only an administrator can unlock the account manually using Locked Users screen.
Apply Account Locking Policy To
Specifies whether the account locking settings apply to all users or all users except the predefined users. Default is All Users except Predefined Users.
Save Changes
Saves the changes you have specified in the Account Locking Settings. Your changes take effect immediately.

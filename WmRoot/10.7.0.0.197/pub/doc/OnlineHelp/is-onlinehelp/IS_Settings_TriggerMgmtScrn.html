webMethods Messaging Trigger Management Screen
Screen Heading: Settings > Messaging > webMethods Messaging Trigger Management
To open the webMethods Messaging Trigger Management screen, click Messaging under the Settings menu on the Navigation panel. On the Messaging Screen, click webMethods Messaging Trigger Management.
Click Return to Messaging to return to the previous screen.
Click View webMethods Messaging Settings to view messaging connection aliases created for the webMethods Broker and Software AG Universal Messaging
Click Edit Global webMethods Messaging Trigger Controls to open the Edit Global webMethods Messaging Trigger Controls Screen and edit the webMethods messaging trigger control details.
Click Refresh Page to update the information on the screen.
Global webMethods Messaging Trigger Controls
Document Retrieval
Current Threads Displays the number of threads currently retrieving documents from the Broker. Each trigger uses a separate thread to retrieve documents. The displayed value does not include threads used to retrieve documents from Universal Messaging.
Maximum Threads Displays the maximum number of server threads that can retrieve documents from the Broker at one time. The default is equivalent to 100% of the server thread pool.
Queue Capacity Throttle Displays the percentage of maximum capacity to which each trigger queue fills. Integration Server adjusts the refill level of each trigger queue by the same amount. For example, if a trigger has a capacity of 10 and a refill level of 4, setting the Queue Capacity Throttle to 50% of maximum reduces capacity to 5 and refill level to 2.
Note:
webMethods messaging triggers that receive documents from Universal Messaging do not have a refill level.
Document Processing
Current Threads Displays the current number of threads that are processing documents for webMethods messaging triggers.
Maximum Threads Displays the maximum number of threads in the server thread pool that can process documents in trigger queues at one time. The default is equivalent to 100% of the server thread pool.
Execution Threads Throttle Percentage of the configured maximum execution threads value at which you want to the server to function. Integration Server applies this percentage to the maximum execution threads value for all concurrent webMethods messaging triggers.
Individual webMethods Messaging Trigger Controls
Trigger Name
Name of the webMethods messaging trigger.
Connection Alias Name
Name of the messaging connection alias used by the webMethods messaging trigger.
Type
Indicates the webMethods messaging provider used with the messaging connection alias. The Type can be Broker or Universal Messaging.
Note:
You can use the watt.server.trigger.managementUI.excludeList server property to filter the list of displayed triggers.
Document Retrieval
Active
Indicates whether document retrieval is active or suspended for this trigger. Yes indicates that document retrieval is active. No indicates that document retrieval is suspended. An asterisk (*) appears next to the status if the document retrieval state is temporary.
To edit the document retrieval state of a specific webMethods messaging trigger, click Yes or No. The Edit State screen for that trigger will be displayed. For more information, see Edit State for Individual webMethods Messaging Trigger Screen.
To edit the document retrieval state for all webMethods messaging triggers, click edit all to open the Edit State for All webMethods Messaging Triggers Screen screen.
Queue Capacity
Specifies the current maximum capacity for the trigger queue. If the current queue capacity is different from the configured queue capacity (that is, the Queue Capacity Throttle is set to less than 100%), the configured queue capacity is displayed in parenthesis.
Current Queue Count
Number of documents currently in the trigger queue on Integration Server.
Document Processing
Active
Indicates whether document processing is active or suspended for this trigger. Yes indicates that document processing is active. No indicates that document processing is suspended. An asterisk (*) appears next to the status if the document processing state is temporary.
To edit the document processing state for a specific webMethods messaging trigger, click Yes or No. The Edit State screen for that trigger will be displayed. For more information, see Edit State for Individual webMethods Messaging Trigger Screen.
To edit the state for all webMethods messaging triggers, click edit all to open the Edit State for All webMethods Messaging Triggers Screen screen.
Processing Mode
Specifies how Integration Server processes the documents in the trigger queue. Serial specifies that Integration Server processes documents in the trigger queue one after the other, in the same order that they were received. Concurrent specifies that Integration Server processes as many documents in the trigger queue as it can at one time. The maximum number of documents the server can process at one time is determined by the Max threads value set for the webMethods messaging trigger.
Maximum Threads
Specifies the maximum number of server threads that can process documents for this webMethods messaging trigger concurrently. If the current maximum threads value is different from the configured maximum threads value (that is, the Execution Threads Throttle is set to less than 100%), the configured maximum threads value is displayed in parenthesis.
Current Thread Count
Specifies the current number of server threads processing documents for this webMethods messaging trigger.
